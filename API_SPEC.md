# API: Анализатор тендерной документации

## Базовый URL
```
https://analyze-final.onrender.com
```

## Вариант 1: Файлы по URL (рекомендуется для Битрикс)

**Запрос:**
```
POST /api/analyze
Content-Type: application/json

{
  "files": [
    "https://example.com/path/to/document.zip",
    "https://example.com/path/to/contract.pdf"
  ],
  "callback_url": "https://your-bitrix.ru/rest/1/webhook-token/bizproc.event.send/",
  "prompt": null
}
```

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `files` | string[] | да | Массив прямых URL для скачивания файлов (PDF, DOCX, DOC, XLSX, XLS, HTML, TXT, ZIP) |
| `callback_url` | string | нет | URL куда сервер отправит POST с результатом после обработки |
| `prompt` | string | нет | Свой промпт. Если не передан — используется стандартный |

**Ответ (мгновенный):**
```json
{
  "task_id": "aef84b03-56cb-4021-8a7e-8a09e8fda61b",
  "status": "processing",
  "message": "Принято 1 файлов. Результат будет отправлен на callback_url."
}
```

## Вариант 2: Загрузка файлов напрямую (multipart)

**Запрос:**
```
POST /analyze
Content-Type: multipart/form-data

files: <бинарный файл>
files: <бинарный файл>
prompt: <текст промпта, необязательно>
```

**Ответ (мгновенный):**
```json
{
  "task_id": "5d78ebb1-29dd-4ac8-aef0-c69a9d3d5d60"
}
```

## Получение результата

### Способ A: Callback (для Битрикс)

Если передан `callback_url` — сервер сам отправит POST на этот URL после обработки (через 2-5 мин).

**Успех:**
```json
{
  "task_id": "aef84b03-...",
  "status": "done",
  "results": [
    {
      "title": "СРОК ПОСТАВКИ",
      "answer": "Период поставки с 1 до 1000 календарных дней",
      "citation": "п. 3.1 — «Период поставки товаров...»"
    }
  ],
  "meta": {
    "files": [{"name": "contract.pdf", "type": "КОНТРАКТ", "chars": 52701}],
    "skipped": [],
    "context_chars": 88128,
    "truncated": true
  }
}
```

**Ошибка:**
```json
{
  "task_id": "aef84b03-...",
  "status": "error",
  "detail": "Текст ошибки"
}
```

### Способ B: Polling (для отладки)

```
GET /status/{task_id}
```

Опрашивать каждые 3-5 секунд. Формат ответа такой же как в callback.

| 1 | СРОК ПОСТАВКИ | Сроки и формулировки |

| 4 | ЕСТЬ ЛИ МОНТАЖ? | Монтаж, пусконаладка |
| 5 | Документы при поставке | Сертификаты, паспорта |
| 6 | Штрафные санкции | Размеры штрафов |
| 7 | Срок оплаты | Дни, условия |
| 8 | Нацрежим | ПП 1875/925, запрет/ограничение/преимущество |

## Важно

- Файлы по URL должны быть напрямую скачиваемыми (не Google Drive, не требующие авторизации)
- Обработка занимает 2-5 минут в зависимости от объёма
- Сервер на бесплатном Render — первый запрос после простоя может занять ~30 сек на пробуждение
- Максимальный размер контекста ~80 000 символов — крупные документы обрезаются по релевантности
- Поддерживаемые форматы: PDF, DOCX, DOC, XLSX, XLS, HTML, TXT, ZIP (включая вложенные)
