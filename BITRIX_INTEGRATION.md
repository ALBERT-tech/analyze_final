# Интеграция с Битрикс24: Анализатор тендерной документации

## Общая схема

```
Менеджер прикрепляет ZIP к сделке
    ↓
Робот: получает URL файла из поля сделки
    ↓
Робот: POST на сервер анализатора (с API-ключом и callback_url)
    ↓
Сервер мгновенно отвечает: {"task_id": "xxx"}
    ↓
... сервер обрабатывает 30-60 сек ...
    ↓
Сервер отправляет POST на callback_url с результатом
    ↓
Битрикс принимает webhook → вставляет текст отчёта в комментарий к сделке
```

## Подготовка в Битрикс24

### 1. Пользовательское поле в сделке
Создать поле типа "файл" — например "Тендерная документация".
Менеджер будет загружать туда ZIP-архив с документами тендера.

### 2. Входящий вебхук
Создать входящий вебхук (или обработчик события) для приёма POST-запроса от сервера анализатора. Этот обработчик должен:
- Принять JSON из тела POST-запроса
- Извлечь поле `external_id` (ID сделки)
- Извлечь поле `text` (готовый текстовый отчёт)
- Вставить `text` в комментарий к сделке по `external_id`

### 3. Робот в сделке
Триггер: заполнено поле "Тендерная документация".

Шаги робота:
1. Получить URL файла из поля сделки (REST API: `disk.file.get` → поле `DOWNLOAD_URL`)
2. Отправить HTTP-запрос на сервер анализатора (см. ниже)

---

## HTTP-запрос от робота

```
POST http://85.239.34.39/api/analyze

Headers:
  Content-Type: application/json
  X-API-Key: <ключ будет предоставлен>

Body (JSON):
{
  "files": ["https://your-bitrix.bitrix24.ru/disk/download/..."],
  "mode": "short",
  "callback_url": "https://your-bitrix.bitrix24.ru/rest/1/<webhook-token>/...",
  "external_id": "DEAL_12345"
}
```

### Параметры запроса

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| files | string[] | да | Массив URL для скачивания файлов. URL должен отдавать файл напрямую (без HTML-страниц и редиректов на логин) |
| mode | string | нет | "short" — обычный анализ (11 вопросов), "full" — расширенный (19 вопросов). Берем : "short" |
| callback_url | string | да | URL куда сервер отправит результат после обработки |
| external_id | string | нет | ID сделки в Битрикс (или любой внешний идентификатор). Возвращается в callback без изменений |

### Ответ (мгновенный)
```json
{
  "task_id": "aef84b03-56cb-4021-8a7e-8a09e8fda61b",
  "status": "processing",
  "message": "Принято 1 файлов. Результат на callback."
}
```

---

## Callback от сервера (через 30-60 сек)

Сервер отправляет POST на `callback_url`.

### Успешный результат
```json
{
  "task_id": "aef84b03-...",
  "external_id": "DEAL_12345",
  "status": "done",
  "text": "АНАЛИЗ ТЕНДЕРНОЙ ДОКУМЕНТАЦИИ\n\n1. НОМЕР И НАИМЕНОВАНИЕ ЗАКУПКИ\n   №91212877 — Поставка краски...\n\n2. ЗАКУПКА ДЛЯ СМП\n   Да\n\n3. НАЦИОНАЛЬНЫЙ РЕЖИМ\n   ПП 1875 — запрет\n\n...",
  "results": [...],
  "meta": {...}
}
```

### Ошибка
```json
{
  "task_id": "aef84b03-...",
  "external_id": "DEAL_12345",
  "status": "error",
  "detail": "Текст ошибки"
}
```

### Ключевые поля callback

| Поле | Описание |
|------|----------|
| external_id | ID сделки — тот же что был передан в запросе |
| status | "done" или "error" |
| text | Готовый текстовый отчёт для вставки в комментарий (простой текст, не HTML) |
| results | JSON-массив с детальными ответами (для парсинга по полям, если нужно) |

**Для вставки в комментарий достаточно поля `text`.**

---

## Содержание отчёта (mode: "short")

1. Номер и наименование закупки
2. Закупка для СМП/СМСП (да/нет)
3. Национальный режим (ПП 1875/925 — запрет/ограничение/преимущество)
4. Срок поставки
5. Место поставки
6. Штрафы и неустойки
7. Срок оплаты
8. Документы при отгрузке
9. Возможность аналога
10. Контакты по техническим вопросам
11. Спецификация (товар — количество — ед. изм.)

Каждый пункт содержит ответ + дословные цитаты с указанием документа и пункта.

---

## Поддерживаемые форматы файлов

PDF, DOCX, DOC, XLSX, XLS, HTML, TXT, ZIP (включая вложенные ZIP с кириллическими именами).

Рекомендация: загружать ZIP-архив со всей документацией тендера.

---

## Обработка ошибок

| Код | Ситуация | Действие |
|-----|----------|----------|
| 200 | Запрос принят | Ждать callback |
| 400 | Нет файлов / ошибка параметров | Проверить body запроса |
| 401 | Неверный API-ключ | Проверить X-API-Key |
| 429 | Дневной лимит исчерпан | Подождать до следующего дня |

Если callback пришёл с `"status": "error"` — показать `detail` менеджеру.

## Тестирование (curl)

```bash
# Отправить файл напрямую
curl -X POST http://85.239.34.39/api/analyze \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <ключ>" \
  -d '{
    "files": ["https://url-файла.zip"],
    "mode": "short",
    "external_id": "TEST_001"
  }'

# Проверить статус вручную (без callback)
curl http://85.239.34.39/status/<task_id> \
  -H "X-API-Key: <ключ>"
```
